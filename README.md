# Blogr Landing Page

A solution to the [Blogr landing page challenge on Frontend Mentor](https://www.frontendmentor.io/challenges/blogr-landing-page-EX2RLAApP).

## Overview

### The Challenge

Users should be able to:

- View the optimal layout for the site depending on their device's screen size
- See hover states for all interactive elements on the page
- Toggle the mobile navigation menu
- Open and close dropdown menus on both desktop and mobile

### What I Learned

- How to build accessible dropdown navigation menus with `aria-expanded` and `role` attributes
- How to use CSS Grid for two-column section layouts
- How to use CSS custom properties (variables) for consistent theming
- How to implement a mobile hamburger menu with JavaScript

### Key Features

- ✅ Fully responsive (mobile 375px → desktop 1440px)
- ✅ Accessible navigation with ARIA attributes
- ✅ Desktop dropdown menus (click to open, click outside to close, Escape to close)
- ✅ Mobile hamburger menu with animated open/close
- ✅ Mobile sub-dropdown menus
- ✅ Hover states on all interactive elements
- ✅ Correct Overpass + Ubuntu font pairing
- ✅ Accurate color palette from the style guide
- ✅ Gradient hero section with decorative circles
- ✅ Dark gradient body section
- ✅ Rounded section corners matching the original design

## Style Guide

| Property | Value |
|---|---|
| Light red (CTA) | `hsl(356, 100%, 66%)` |
| Very light red (hover) | `hsl(355, 100%, 74%)` |
| Very dark blue (headings) | `hsl(208, 49%, 24%)` |
| Very dark grayish blue (body) | `hsl(207, 13%, 34%)` |
| Very dark black blue (footer) | `hsl(240, 10%, 16%)` |
| Hero gradient | `hsl(13, 100%, 72%)` → `hsl(353, 100%, 62%)` |
| Body gradient | `hsl(237, 17%, 21%)` → `hsl(237, 23%, 32%)` |
| Font (Headings) | Overpass 300, 600 |
| Font (Body) | Ubuntu 400, 500, 700 |
